容器成员函数

std::vector::value_type 其中value_type是vector容器的成员类型，在std::vector中value_type是int的别名，其他类型同理

成员类型定义了成员函数的“输入参数”和“返回值”的规格，是数据交换的媒介

  • 屏蔽底层细节： 无论底层是数组、链表还是树，你都可以统一用 container::iterator 来遍历，而不需要关心它具体是一个原始指针还是一个复杂的对象。
  • 增强代码可移植性： 比如 size_type，在 32 位系统上可能是 unsigned int，在 64 位系统上是 unsigned long long。通过使用成员类型，你的函数签名永远是 size_type size()，代码不需要修改。
  • 支持泛型算法： 标准库中的算法（如 std::sort）通过读取这些成员类型，就能知道该如何处理这个容器。

注：
统一接口： 很多标准库算法（如 std::accumulate）并不直接拿 T，而是通过访问容器内部的 value_type 来保持一致性。
类型变换： 有时候 value_type 可能和模板参数 T 不完全一致（虽然在 vector 中它们是一样的，但在某些特殊容器里可能会有细微差别）。
可读性： value_type 明确指代“这是容器里存的东西”，而 T 只是一个抽象的符号。

TIPS

  • typename用在定义模板的代码中，指定 依赖于模板参数的类型名称 是一个类型名而非编译器默认指定的静态成员变量/常量类型（值类型）。

  • 不应使用noexcept关键字：含有分配内存、打开文件功能的函数

  • 移动构造函数的作用就是转移某一处资源的使用权

    std::move、右值引用、移动构造函数 三者协同合作实现移动语义

std::vector<>对比std::valarray<>

std::vector是动态通用序列容器，支持快速随机访问和动态扩容，适用于通用数据存储；std::valarray是专为数值计算优化的数组类，不支持动态调整大小，但支持高效的切片、掩码操作和逐元素数学运算。二者区别在于用途、内存管理及数值计算能力。

核心区别对比：

  1. 功能定位：
    • std::vector (STL): 泛型容器，适用于存储、插入、删除任意类型数据，常用于管理动态数组。
    • std::valarray (数值库): 专注于数值数组，不属于STL容器，不支持 push_back，适用于科学计算。
  2. 大小调整：
    • std::vector: 支持 resize()、push_back()，运行时动态增加或减少容量。
    • std::valarray: 大小固定（或者需要重新赋值），无法动态扩容。
  3. 数值运算能力：
    • std::vector: 需要使用循环或 std::transform 进行逐元素计算。
    • std::valarray: 提供了直接的逐元素数学运算符（如 v1 + v2、v1 * 2.0），且运算速度可能通过编译优化更快。
  4. 切片与子集操作：
    • std::valarray: 强项，支持 slice (切片)、gslice (广义切片)、mask (掩码) 和 indirect (间接) 访问。
    • std::vector: 原生不支持，需要自行编写逻辑或使用STL算法。

总结使用场景：

  1. 需要灵活大小、插入删除数据时，选择 std::vector。
  2. 需要进行复杂的数学运算（如矩阵加减、向量乘法）、数据分析时，选择 std::valarray。

类成员访问权限的决定

  • 默认设定其访问权限为private,按需设定为public、protected

private：数据 + 实现细节

满足任意一条就应设定为private：

  1. 成员表示对象的状态

  2. 成员的值需要保持某种约束/不变量

  3. 外部不需要也不应该知道它存在

public：稳定的“能力接口”

一个成员能是 public，必须同时满足：

  1. 语义稳定（不会频繁或大量更改）

  2. 代表“能做什么”而不是“怎么做”

  3. 不暴露内部结构

  4. 不会破坏类的不变量

protected：专门给子类的通道

明确打算让子类扩展行为，但又不想对“所有外部用户”开放时设定。
